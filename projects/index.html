Project 01 · 2026
MeetWise — AI Scheduling Platform
Node.js Cloudflare Workers Claude API Google OAuth2
View Live Site

Overview

MeetWise is a production AI scheduling platform that automates calendar conflict detection and meeting booking. It integrates Google OAuth2 and freeBusy APIs for real-time availability, routes AI calls through a Cloudflare Worker proxy to Claude API, and pulls live weather data from Open-Meteo to generate context-aware itineraries. Scheduling time reduced from ~10 minutes to under 1 minute.

Architecture

User Login→Google OAuth2→freeBusy API
Availability Slots + Weather→Cloudflare Worker→Claude API

Key Engineering Decisions

▸ Cloudflare Worker as proxy: Keeps the Claude API key off the client entirely. All AI requests are validated and forwarded server-side at the edge with no cold start latency.
▸ freeBusy API for privacy: Only free/busy time blocks are queried — not full event details — giving the AI enough to find open slots without exposing private calendar data.
▸ Weather-aware itineraries: Open-Meteo forecast data is injected into the Claude prompt so suggestions account for indoor vs. outdoor conditions on the meeting day.
Project 02 · 2026
Instagram Snake Detector
JavaScript File API Instagram
View Live Site

Overview

Instagram Snake Detector identifies "snakes" — people you follow on Instagram who don't follow you back. The user drags and drops their Instagram data export files (followers_1.html and following.html) into the UI, which parses them entirely client-side, computes the difference, and displays the full snake list instantly.

The tool then generates a ready-to-run JavaScript console script the user can paste into Instagram's browser DevTools to automatically unfollow every snake — no manual clicking through hundreds of profiles required. No backend, no server, no data leaves the device.

How It Works

01 Export Instagram data: The user downloads their data archive from Instagram settings under "Your Activity > Download your information", which includes followers_1.html and following.html.
02 Drag and drop: Both files are dragged into separate drop zones in the UI. The browser's File API reads them locally — nothing is uploaded to any server.
03 Snake detection: Both HTML files are parsed to extract usernames. A set difference is computed — anyone in following but not in followers is flagged as a snake and shown in the list.
04 Console script: A JavaScript snippet is generated for the user to copy and paste into Instagram's browser console, which automates unfollowing each snake without touching any private API.

Core Logic

// Parse usernames out of Instagram's exported HTML file
function parseUsernames(htmlText) {
  const doc = new DOMParser().parseFromString(htmlText, 'text/html');
  return new Set(
    [...doc.querySelectorAll('a')].map(a => a.textContent.trim())
  );
}

// Snakes = people you follow who don't follow back
const snakes = [...following].filter(u => !followers.has(u));

// Display results and generate the unfollow console script
displaySnakes(snakes);
generateScript(snakes);

Privacy

100% client-side — no server, no database, no accounts needed. Files are read locally and never transmitted. The generated script runs directly against Instagram's web UI using no third-party APIs or credentials.

2048 AI Solver
Python NumPy Minimax Heuristic Search

Overview

An autonomous solver for 2048 using the Expectimax algorithm — a Minimax variant for stochastic environments. The AI maximizes score while random tile spawns are modeled as chance nodes computing expected values. Validated across 10,000+ simulation runs with a >98% win rate.

Heuristics

01Empty Tiles: More empty cells = more future moves. Logarithmically weighted.
02Monotonicity: Values decrease consistently in one direction, keeping the largest tile in a corner.
03Smoothness: Penalizes large value differences between adjacent tiles.
04Corner Priority: Bonus if the highest tile is in a corner, enabling better merge chains.
def evaluate(board):
    empty  = count_empty(board) * 2.7
    mono   = monotonicity(board) * 1.0
    smooth = smoothness(board)  * 0.1
    corner = max_corner_bonus(board) * 1.0
    return empty + mono + smooth + corner
